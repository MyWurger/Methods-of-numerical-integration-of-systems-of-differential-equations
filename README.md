# Практическое занятие №1

## Методы численного интегрирования систем дифференциальных уравнений. Моделирование движения КА

Проект представляет собой настольное приложение на `C++17` и `Qt 6`, моделирующее движение космического аппарата в центральном гравитационном поле Земли. Приложение позволяет задавать начальные условия, выбирать численный метод интегрирования, запускать расчёт и анализировать результат в виде графиков и таблицы.

## Что умеет программа

- моделирует движение КА в абсолютной геоцентрической системе координат;
- использует два метода численного интегрирования:
  - метод Эйлера;
  - метод Рунге-Кутты 4 порядка;
- позволяет задавать:
  - начальные координаты `x0, y0, z0`;
  - начальные скорости `Vx0, Vy0, Vz0`;
  - время начала `t0`;
  - терминальный момент `tk`;
  - шаг интегрирования `h`;
- отображает результаты в виде:
  - графиков `X(t)`, `Y(t)`, `Z(t)`;
  - графиков `Vx(t)`, `Vy(t)`, `Vz(t)`;
  - траекторий в плоскостях `XY`, `YZ`, `XZ`;
  - таблицы рассчитанных точек;
- поддерживает масштабирование и панорамирование графиков;
- останавливает моделирование при столкновении с Землёй;
- позволяет экспортировать таблицу результатов в `PNG`.

## Используемая модель движения

В проекте реализована модель движения центра масс КА в центральном гравитационном поле Земли:

```text
x'  = Vx
y'  = Vy
z'  = Vz
Vx' = -mu * x / r^3
Vy' = -mu * y / r^3
Vz' = -mu * z / r^3

r = sqrt(x^2 + y^2 + z^2)
mu = 3.98603 * 10^14 м^3 / с^2
```

Если расстояние до центра Земли становится меньше либо равно радиусу Земли, расчёт останавливается как столкновение.

## Архитектура проекта

Основные классы:

- `TDynamicModel` — абстрактный интерфейс модели правых частей системы ОДУ;
- `TSpaceCraft` — конкретная модель движения КА;
- `TAbstractIntegrator` — базовый класс численного интегратора;
- `TEuler` — реализация метода Эйлера;
- `TRungeKutta` — реализация метода Рунге-Кутты 4 порядка;
- `TVector` — вспомогательный класс вектора состояния;
- `MainWindow` — главное окно приложения;
- `PlotChartWidget` — виджет построения и интерактивного просмотра графиков.

## Требования

Для сборки и запуска нужны:

- `CMake >= 3.16`
- компилятор с поддержкой `C++17`
- `Qt 6` с модулями:
  - `Widgets`
  - `Svg`
  - `Charts`
- `macOS`

Проект ориентирован на запуск на `macOS`. Сценарий сборки автоматически ищет типовые установки `Qt` на Mac и формирует `.app bundle`.

## Сборка

### Вариант 1. Через preset

Если используется preset из проекта:

```bash
cmake --preset default-debug
cmake --build --preset default-debug
```

### Вариант 2. Обычная сборка

```bash
cmake -S . -B build/default-debug
cmake --build build/default-debug
```

## Запуск

После сборки приложение можно запускать как обычный `macOS` bundle:

```bash
open build/default-debug/ModelingMethodsLab1.app
```

Либо напрямую бинарник:

```bash
./build/default-debug/ModelingMethodsLab1.app/Contents/MacOS/ModelingMethodsLab1
```

## Использование

1. Ввести начальные координаты и скорости КА.
2. Выбрать метод интегрирования.
3. Задать `t0`, `tk` и `h`.
4. Нажать `Выполнить моделирование`.
5. Просмотреть:
   - вкладку `Координаты`;
   - вкладку `Скорости`;
   - вкладку `Траектории`;
   - вкладку `Таблица`.

Если расчёт дошёл до поверхности Земли, на координатных графиках будет показана отметка события `|r| = Rземли`.

## Экспорт таблицы

После успешного расчёта становится доступна кнопка `Сохранить PNG`.  
Изображение таблицы сохраняется в каталог:

```text
exports/tables
```

Имя файла формируется автоматически по текущей дате и времени.

## Особенности реализации

- данные графиков строятся на основе общего типа `SimulationResult`;
- таблица работает через модель `QAbstractTableModel`;
- графики поддерживают:
  - zoom;
  - pan;
  - пользовательские отметки событий;
- для `macOS` после сборки выполняется:
  - деплой Qt-фреймворков;
  - копирование нужных плагинов;
  - ad-hoc подпись `.app bundle`.

Учебный проект по дисциплине, посвящённой методам моделирования и численного интегрирования систем дифференциальных уравнений.
